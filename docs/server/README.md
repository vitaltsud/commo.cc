# Сервер: Kontabo

## Хост

| Параметр    | Значение      |
|------------|---------------|
| IP         | `84.247.164.76` |
| Хостинг    | Kontabo       |

На этом же сервере работают **отдельные проекты**. При настройке commo.cc их не трогать:
- **n8n** — https://ai.crmca.cc/ (свой vhost, свой порт).
- **commo.cc** — маркетплейс (отдельный vhost, порт 3000).

**Доступ только по доменам:** доступ по IP запрещён.
- **Nginx:** default server ([nginx-00-default-drop.conf](nginx-00-default-drop.conf)) отдаёт `return 444` на запросы по IP или с неизвестным Host.
- **Файрвол (ufw):** открыты только 22 (SSH), 80 (HTTP), 443 (HTTPS). Порт 3000 (приложение) снаружи закрыт — доступ только через nginx по домену **https://commo.cc**.

## SSH

- **Доступ:** по ключам (парольный вход отключён или должен быть отключён).
- **Проверка:** `ssh -i <path-to-key> user@84.247.164.76`

Перед любыми изменениями:
- Не менять существующие конфиги веб-сервера/систем — только **добавлять** новые (vhost, сервисы). В т.ч. не трогать конфиг для ai.crmca.cc (n8n).
- Не перезаписывать общие файлы (nginx.conf, apache2.conf) без бэкапа и проверки.
- Сначала снимать бэкапы и тестировать на отдельном vhost/порту.
